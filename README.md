# HW1

強化學習網格世界系統 (RL GridWorld System)本專案實作了一個基於強化學習（Reinforcement Learning）理論的網格世界演示系統。透過三個獨立的階段，展示了從環境建模、策略評估到最優路徑規劃的完整流程。📖 功能描述本系統分為三個主要部分，分別對應強化學習中的核心概念：1. H1-1: 網格地圖開發 (Environment Setup)本階段定義了機器人運作的「環境」。自主配置：使用者可自定義網格大小（$N \times N$），並手動設定起點、終點。障礙物規則：支援設定 $N-2$ 個障礙物。這些障礙物在後續演算法中會被視為「不可通行」的狀態，機器人必須繞過它們才能抵達終點。2. H1-2: 隨機生成行 (Policy Evaluation)本階段旨在評估「當機器人沒有目標，只是在環境中亂走」時，地圖上每一格的預期價值。隨機策略：模擬機器人在每一格往四個方向移動機率皆為 $25\%$ 的情況。價值計算：透過迭代計算出每一格的價值數值（Value）。數值的高低反映了在隨機移動下，從該位置抵達終點平均需要付出的步數代價。動態特性：每次點擊執行時，系統會隨機分配權重，讓使用者觀察不同隨機策略組合下價值矩陣的變化。3. H1-3: 使用價值迭代算法推導最佳政策 (Value Iteration)本階段是系統的核心，旨在找出從起點到終點最節省步數的最佳方案。最優化運算：使用價值迭代演算法（Value Iteration），透過貝爾曼最優方程尋找每一格的最優行動（Optimal Policy）。路徑與箭頭一致性：系統會自動計算出一條綠色標記的最佳路徑。為確保視覺呈現與邏輯一致，路徑格中的箭頭會精確地指向下一個移動目標，形成清晰的階梯狀導航圖。無障礙展示：預設展示在無障礙物情況下，算法如何高效地規劃出最短路線。🛠 技術實作後端架構：使用 Flask 框架處理 API 請求。核心算法：利用 NumPy 進行矩陣運算，實作策略評估與價值迭代。路徑追蹤：在價值迭代後，透過對比相鄰格子的價值大小，自動追蹤並提取路徑序列。前端視覺化：使用原生 JavaScript 動態生成網格並同步顯示運算結果。📦 快速啟動安裝必要套件：Bashpip install flask numpy
